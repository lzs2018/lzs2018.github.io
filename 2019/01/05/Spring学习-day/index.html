Spring框架学习-day01

sping负责管理项目中的所以对象，spring看作是项目中对象的管家，是一个一站式框架，容器中装什么对象就有什么功能

Spring 的核心是控制反转（IoC）和面向切面（AOP）

EE 开发分成三层结构:

  1. WEB 层:Spring MVC.
  2. 业务层:Bean 管理:(IOC)
  3. 持久层:Spring 的 JDBC 模板.ORM 模板用于整合其他的持久层框架.

不排斥其他框架，还能帮助其他框架管理对象，就是一个容器

IOC和DI

IOC:反转控制
将我们创建对象的方式反转
以前创建对象的方式由我们开发人员自己维护，包括依赖关系也是自己注入
使用sprin之后，对象的创建和依赖关系可以由spring创建和注入
反转创建就是反转了对象创建的方式，从我们自己创建反转给了程序

DI:依赖注入
实现ioc思想 需要我们的DI支持

注入方式：set方式注入
1.构造注入
2.属性直接注入（字段注入）

注入类型：引用注入 (对象注入）
值类型注入（八大基本类型）

AplicationContext

BeanFactory接口，原始接口 针对原始接口功能实现比较单一
特点：每次获取对象时才会创建对象
AplicationContext：每次容器启动时，就会创建容器中配置的对象，体现了丰富的功能

从类路径下加载配置文件：ClassPathXmlApplicatCation

从硬盘绝对路径下加载配置文件：FileSystemXmlApplicationContext

配置详解

Bean元素

使用该元素描述需要spring容器管理对象
name:给管路对象起一个名字，获得对象时根据该名字获得类型 名称可以重复 可以使用特殊字符
class属性：被管理对象的完整类名
id：与name属性一样，不能使用特殊字符 名称不可字符

结论：尽量使用name 属性
